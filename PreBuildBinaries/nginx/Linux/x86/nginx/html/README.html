Comments or Issues:

FLVPlayback and Flex Video components require custom module

The Flash FLVPlayback component and Flex video component require a server side module be included in your application's configuration to operate properly. Many open source media player components embed the FLVPlayback component. To get this component to work with the Wowza Pro server, follow the steps below to add the needed module definition to your application. For this example, I will configure an application named mystreamingvideos and will use a .flv file name of mycoolvideo.flv.

  1. Create an application folder [install-dir]/applications/mystreamingvideos.
  2. Create a configuration folder [install-dir]/conf/mystreamingvideos.
  3. Copy [install-dir]/conf/Application.xml into the new configuration folder.
  4. Edit [install-dir]/conf/mystreamingvideos/Application.xml and add the following module definition to the end of the <Modules> list: 
    
<Module>
	<Name>FLVPlayback</Name>
	<Description>FLVPlayback</Description>
	<Class>com.wowza.wms.module.ModuleFLVPlayback</Class>
</Module>
  5. At this time you might also want to specify a folder path for Streams/StorageDir. If this value is set, the server will serve .flv files from the specified folder. If left unset, the server will serve .flv from the default folder: [install-dir]/applications/mystreamingvideos/streams/_definst_
  6. Copy the mycoolvideo.flv into the appropriate folder as specified in the previous step.
  7. In the Flash editor, create an instance of the FLVPlayback component on the stage, open the component inspector and change the contentPath to rtmp://localhost/mystreamingvideos/mycoolvideo.flv.
  8. Select Control>Test Movie from the Flash menus.

Connecting over RTMPT

The default Server.xml file is configured to listen on port 1935 only (the default rtmp port). To connect to the server over rtmpt you can either uncomment the port 80 configuration section in the Server.xml file or connect over rtmpt using port 1935 (example: rtmpt://localhost:1935/applicationname).

Memory Settings

If you are running into OutOfMemory errors while running the server you may want to increase the maximum size of the Java heap. By default the Java maximum heap size is set at 768MB. Consult the Runtime Configuration section of the Server Administration chapter in the user guide for the location and names of the configuration file and settings.

Performance Tuning - Linux

Wowza Media Server Pro uses an asynchronous file io scheduling algorithm that smoothes out the file io performance. This systems works very well in tandem with the io scheduler (either the deadline or anticipatory as scheduling algorithm works well) that is part of the Linux kernel. By default many Linux distributions have the io scheduler turned off. To turn it on consult your Linux distributions documentation. For Red Hat the io scheduler can be turned on as part of the kernel parameters at startup. If you are using the grub boot loader, you can edit the /boot/grub/grub.conf file and add the elevator=as or elevator=deadline parameter to the kernel statement.

Windows Service

By default Wowza Media Server Pro Windows service is set to Manual startup. This is so it does not conflict with the version started from the command prompt. Consult the Starting and Stopping the Server section of the Server Administration chapter in the user guide for more information on how to change this to Automatic startup.

JMX and the Windows Service

When running Wowza Media Server Pro as a Windows service, the JMX interface will not be available unless the service is running as a named user. To configure the service to run as a named user, go to Settings>Control Panel>Administrative Tools>Services and right click on the Wowza Media Server Pro service and select Properties. Next, click on the Log On tab, change the Log on as radio to This account and enter a user name and password for a local user.

OS X Service

By default the com.wowza.WowzaMediaServerPro OS X service is set to Disabled startup. This is so it does not conflict with the version started from the command prompt. To enable the launchd service to startup automatically when the machine is restarted issue the following command in a terminal window:

sudo launchctl load -w /Library/LaunchDaemons/com.wowza.WowzaMediaServerPro.plist

To disable:

sudo launchctl unload -w /Library/LaunchDaemons/com.wowza.WowzaMediaServerPro.plist

Debugging

To turn on debug logging for the server, edit the conf/log4j.properties file and change the log level from INFO to DEBUG.

Error: "Too many files open"

If you encounter the Too many files open error on Linux you may need to increase the number of simultaneous files that can be openeded. Please consult your Linux documentation on how to do this for your Linux distribution.

For Red Hat add the following line to your /etc/sysctl.conf

fs.file-max=300000

You also may need to up the ulimit in the command shell that is being used to launch Wowza Media Server Pro. This needs to be done each time the server is invoked from a new command shell. The command to view the current limit is ulimit -n. To set the limit enter the command ulimit -n 2048.

Relocating Wowza Media Server Pro

The Mac OS X and Linux installers do not provide an option to choose the location of the installation of the base server files. For Mac OS X this location is /Library/WowzaMediaServerPro-1.1.0 and for Linux it's /usr/local/WowzaMediaServerPro-1.1.0. All the server startup scripts locate the server files through a symbolic link named /Library/WowzaMediaServerPro (Mac OS X) and /usr/local/WowzaMediaServerPro (Linux). If you wish to relocate the base server files you can move them where you choose and update this link using the ln -sf command.

Error: "Address already in use"

This could be due to several issues

  • If you currently have Flash Media Server running it will conflict with Wowza Media Server Pro. You must stop FMS before starting WMS.
  • If after shutting down the server you try to restart the server and you get the error messages. There are two possible solutions:
    • Edit Server.xml and change all your HostPortList/HostPort/IpAddress entries to the actual ip address of the machine (the default is *).
    • Wait 1-5 minutes to let the underlying TCP sub-system timeout.

applications/[application] Folder Required For Client Connection

The Wowza Pro server now requires that a folder with the same name as the application be presented in the applications folder, to enable client connections to that applications. For example if the client connection url is rtmp://localhost/myapplication, then there must be a folder in the applications folder named myapplication.
